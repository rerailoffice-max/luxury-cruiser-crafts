# GAS セットアップ手順

KOGEI CODE LP のフォーム送信を受け取る Google Apps Script のデプロイ手順です。

---

## 1. Google スプレッドシートを作成

1. [Google スプレッドシート](https://sheets.google.com) で新規スプレッドシートを作成
2. 名前を「KOGEI CODE お問い合わせ」などに変更

## 2. Apps Script エディタを開く

1. メニュー「拡張機能」→「Apps Script」を選択
2. エディタが開く

## 3. コードを貼り付け

1. デフォルトの `Code.gs` の中身を削除
2. `lp/gas/Code.gs` の内容をすべてコピーして貼り付け
3. `CONFIG` の以下の値を書き換える：
   - `ADMIN_EMAIL` → フォーム通知を受け取るメールアドレス
   - `DOCUMENT_URL` → 資料請求者に送る PDF 等の URL

## 4. Web アプリとしてデプロイ

1. 右上の「デプロイ」→「新しいデプロイ」
2. 種類の歯車アイコンをクリック →「ウェブアプリ」を選択
3. 以下を設定：
   - **説明**: `KOGEI CODE フォーム受信 v1`
   - **次のユーザーとして実行**: `自分`
   - **アクセスできるユーザー**: `全員`
4. 「デプロイ」をクリック
5. アクセスを承認（Google アカウントの認証画面が出る場合は許可）
6. 表示される **Web アプリの URL** をコピー

## 5. LP に URL を設定

コピーした URL を LP の環境変数に設定します。

### ローカル開発

`lp/.env.local` に追記：

```
NEXT_PUBLIC_GAS_URL=https://script.google.com/macros/s/xxxxx/exec
```

### Vercel

Vercel ダッシュボード → プロジェクト → Settings → Environment Variables に追加：

| Key | Value |
|-----|-------|
| `NEXT_PUBLIC_GAS_URL` | コピーした Web アプリ URL |

## 6. 動作確認

1. LP のフォームからテスト送信
2. 確認項目：
   - スプレッドシートに行が追加されていること
   - 管理者メールが届くこと
   - 送信者に自動返信メールが届くこと

---

## トラブルシューティング

| 症状 | 対処 |
|------|------|
| CORS エラー | GAS の「アクセスできるユーザー」が「全員」になっているか確認 |
| メールが届かない | GAS エディタの「実行」→「ログを表示」でエラーを確認 |
| スプレッドシートに記録されない | GAS がスプレッドシートに紐づいているか確認（スプレッドシートから開く） |

## GAS コードの更新

コードを修正した場合は、再度「デプロイ」→「デプロイを管理」→ 鉛筆アイコン →「バージョン: 新しいバージョン」→「デプロイ」で更新。
